Memory Puzzler UX Iteration

by Michael Andryauskas

Introduction

For Udacity’s Virtual Reality Nanodegree course, we were tasked to create, test, and iterate on a small memory game on a mobile VR Device from August 15th to September 11th. The instructors provided the initial Unity File, which included art assets and some basic game code. The game was tested using a Samsung Galaxy S7 and a GearVR headset.


Gameplay Screenshot of the Final Game

Gameplay Inspirations

Memory Puzzler uses the same primary mechanic as Simon; the game presents a user with a pattern, and the user needs to repeat the pattern that was presented to him. For this iteration of the app, there is only one pattern of five elements that the user needs to copy to complete the game. The app presents the user with the same pattern until the user correctly repeats the pattern.


Personal Goals

In addition to the primary objective of the course which was to learn about UX in VR Development, I wanted to explore the potential for the app to be used for mental capacity enhancement. I wanted to set the tone of the application to be more clinical in nature rather than mysterious as the initial files suggested. As such, I created my own placeholder environment art assets using Autodesk Maya and Adobe Photoshop rather than the provided art assets.


Process

Statement of Purpose

Memory Puzzler is a VR app for the GearVR that allows those who wish to improve their mental capabilities to refine their memory and concentration.


Persona

For the Persona, I thought about how a user could have a mental disability yet still have an affinity for more high-tech experiences. I created Sam, my persona, who would serve as a reference for design decisions I made. It should be noted that this persona is not fully realized, and a more refined persona would require speaking to more than one potential user who fits the app’s target audience.

User Testing

User Test 1

The main focus of the first test is to determine if the scale of the room was appropriate for users. I relied more on the Unity units for the height rather than my own judgement, since I was using art assets I built myself and I am much taller (6’5”) than the average person. I was only able to test the experience with one person at the time, but she claimed that the scale was only a little smaller than she felt was correct. Since she was around average height, I scaled the room to be a tad larger.



The initial layout of the room from the user's perspective

User Test 2

For the second test, I wanted to verify if the messages for the starting screen and ending UI panels were comfortable to read and to interact with, as well as user’s interpretations of the instruction on each panel. I was only able to have access to one user, and she was able to interact with the buttons, despite this being her first experience interacting with the GearVR headset. In addition, she was able to understand what the instructions were trying to convey, even if she was not sure what it meant by “Spheres.” I altered the wording a little to make it clearer that the spheres were behind the panel.



The first versions of the Worldspace Panels

User Test 3

For the third test, I wanted to determine if the movement speed of the avatar was comfortable for users. I had a participant move from outside the room to the spheres, then from the spheres to the back of the room. She did not feel any motion sickness while being moved throughout the world, nor did she feel the movement took too long. As such, the movement speed did not need to be altered. However, the user did feel uncomfortable moving through the spheres as she moved to the back of the room. In the next iteration of the application, I made the spheres invisible once the user completed the puzzle so the user didn’t move through them anymore.


User Test 4

For the fourth test, I tested the audio feedback and the primary interaction methods with two participants. Both users seemed to have difficulty understanding the initial instructions, and tried to interact with the orbs as the pattern was being displayed. Users also had issues understanding when they entered an incorrect pattern, as there was only audio feedback letting them know they failed.



A screenshot of the user attempting to repeat the pattern.

In the next build, I included a message box under the spheres that gave text feedback to the user. I also made the spheres blink red when the user enters the wrong sphere pattern. Finally, I changed the wording on the first panel to tell the user he needs to observe the pattern the spheres made first before the user could try to repeat the pattern he observed.


Breakdown of the Final Piece

Starting Screen

The user begins in a small compartment that resembles an elevator. In front of the user is a worldspace UI that tells the user how to play the game. They need to observe the pattern the spheres create, then they tap the spheres to repeat the pattern. If the user presses the “Play Game” button, the user’s avatar moves forward and the user can play the game.


The starting panel of the app.

Gameplay

The user is presented with five spheres. Once the user’s avatar is close enough to the spheres, they start to blink in a pattern. The user would then tap a sphere in an attempt to repeat the pattern the spheres made. There is also a messaging box that presents the user with cues telling the user when to observe the pattern, the progress he made in recreating the pattern, and if the user needs to rewatch the pattern. When the user looks directly at a sphere, the sphere turns green and the reticle becomes a ring rather than a circle. If a pattern is entered incorrectly, the spheres blink red to indicate that a wrong sphere was pressed, and the user needs to rewatch the pattern and try again. If the user is able to recreate the pattern, a success sound plays and the spheres disappear and the user moves forward to the end of the room.


The app is showing the user the pattern to repeat.

Ending Screen

The user’s avatar approaches the elevator in the back of the room and a congratulations screen appears in the back of the elevator. This screen also includes a button that, if the user interacts with it, moves the user back to the starting point of the experience.


The user has completed the puzzle!

Conclusion

User Feedback

One encouraging comment I had was that one participant, who had mental handicaps due to a recent car accident, stated that the experience is reminiscent of the paper-based exercises the brain therapist would have him perform to improve his mental capabilities. This user relayed the experience multiple times to try to improve his memory utilizing memory techniques he learned from his brain therapist. From his feedback, I feel like the app would indeed have potential to serve as a mental capabilities recover application.


Next steps

If I was to continue creating this app, I would include a way to have longer and longer patterns for the user to repeat. Longer patterns would give users a goal to not only strive for, but to improve upon. I would also research how brain specialist help patients regain their mental abilities and see how I would integrate their methods into my application. Finally, I would involve more participants in the iterative testing to get a larger sample size so the application would be more accurate in expectations for the spectrum of target audience members.
